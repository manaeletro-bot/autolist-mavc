# 📓 Registro Vora DOC 005 - 19/07/2026

**Versão Correspondente:** Versão 1.6 - Compatibilidade PWA (Progressive Web App)

## 🎯 Objetivo da Sessão
Configurar o projeto AUTOLIST - MEVC para funcionar como uma PWA (Progressive Web App) completa, tornando-o instalável em celulares e desktops, além de atualizar todos os ícones da aplicação com base no arquivo `image/ICOM.png` fornecido pelo usuário.

## 🛠️ Alterações Efetuadas
* **[NEW]** `public/manifest.json`: Manifesto da aplicação PWA especificando metadados, escopo, modo de visualização autônomo (standalone), cores do tema e os caminhos dos novos ícones.
* **[NEW]** `public/sw.js`: Service Worker com cache dinâmico de recursos estáticos essenciais por meio de estratégia Stale-While-Revalidate, permitindo o carregamento imediato da interface e prevendo bypass de cache em requisições de API (`/api/*`).
* **[NEW]** `public/icon-192.png`, `public/icon-512.png` e `public/favicon.png`: Cópias em alta resolução derivadas de `image/ICOM.png` para servirem de ícones oficiais do PWA e favicon da aba superior do navegador.
* **[MODIFY]** `index.html`: Substituição do favicon antigo em formato SVG pelo favicon oficial em formato PNG, inclusão do link de manifesto, meta tag `theme-color` e meta tags de suporte a PWA em dispositivos iOS (Apple mobile web app capable).
* **[MODIFY]** `src/main.jsx`: Inclusão do código nativo para registro condicional do Service Worker (`sw.js`) ao carregar a página no navegador.

## 🎨 Decisões de Layout e Design (UX/UI)
* **Coesão Cromática:** O manifesto e a cor de tema da barra de status móvel foram configurados com `#0ea5e9` (sky-500) para harmonizar perfeitamente com os acentos da marca AUTOLIST - MEVC.
* **Ícone Premium:** O ícone original `ICOM.png` foi distribuído tanto como ícone de exibição nativo do aplicativo nas telas iniciais do dispositivo móvel quanto como favicon do browser.

## 🔳 Testes e Validação Manual
* **Validação de Compilação:** Executado o comando `npm run build`, concluído com êxito (código 0), gerando todos os arquivos estáticos copiados na pasta `dist/`.
* **Validação de Registro:** Utilizado o navegador para confirmar que o Service Worker registra com sucesso no escopo local `http://localhost:3000/` e que a modal de instalação do app fica disponível no menu de extensões do Chrome/Edge.
